This section takes search terms from a file, url encodes them, and makes a request to the search API for each search term, decodes json into a python dictionary, then stores it in output variable. It also stores each pull in a text file. terms: List of all search terms. Pulled from Terms.txt above. url: stores the beginning part of url used to request from the API
rawOutput is a list of lists, each containing a list of the dictionary of results for each search term.
for loop 1: Each call can only collect 100 tweets, and Twitter rate limits to 180 calls per 15 minute period. The first for loop runs every 15 minutes and 5 second to stay within the rate limit. It runs for about five hours.
for loop 2: Loops 7 times so that I make 180 calls within every 15 minute period.
for loop 3: Loops through each search term.
useURL is the full url with the search term encoded. count requests 100 tweets, max_id is a way to index so that each call pulls the next 100 tweets from the search results.
client.request makes the actual request to the Twitter API, using the credentials stored in client. This is done withing a for loop so that I can retry when connection errors occur, but not so many times as to get my IP blacklisted by Twitter.
Each request is saved in a text file, and is written to rawOutput which is the main data object used by the script.
Note: the original dataset only had 5 attributes per tweet, but I am collecting all the attributes available in a Tweet.

url='https://api.twitter.com/1.1/search/tweets.json?'
rawOutput=[]
max_ids=[]
iteration= 0
for period in range(0,20):
    for rep in range(0,int(math.floor(180/len(terms)))):#7 times per 15min
        termCount=0
        for index, term in enumerate(terms):
            if iteration==0:#don't have a max_id on the first iteration
                urlUse=(url+urllib.urlencode({'q' : term})+'&count='+str(100))
                rawOutput.append([])
                max_ids.append([])
            else:
                urlUse=(url+urllib.urlencode({'q' : term})+\
                        '&count='+str(100)+'&max_id='+str(max_ids[termCount]-1))
            for i in range(0,4): #to catch and retry when there are errors
                try:
                    pull=client.request(urlUse)[1]#collect the data
                except Exception:
                    print 'error'
                    time.sleep(15)
                    pass
                else:
                    break
            fileName = "{}_{}_{}.txt".format(str(period),str(rep),str(index))
            with open(workingDir + "\pulls\\" + fileName, "w") as file:
                file.write(pull)
            rawOutput[termCount]+=[json.loads(pull)]
            #Remember the last tweet in max_id:
            max_ids[termCount]=rawOutput[termCount][iteration]['search_metadata']['max_id']
            termCount+=1
        iteration+=1
    time.sleep(15*60+5) #wait fifteen minutes
print 'done'

Test completeness of collected data
Tweets contain many attributes that can be null. I will check for the presence of several attributes of interest. Since these attributes are generated by the api, they are likely to be valid if present.
When the tweet is missing, meaning there were no more relevant tweets provided by the api, all fields are recorded as missing.
0) contributor id: the unique id for a given user. Should be an int or long.
1) coordinates: location, longitude then latitude of tweet origin. Check if null
2) hashtags: list of hashtags on a tweet. Check if empty. Empty does not mean an error, but this output will give a sense of how many have no hashtags to work with.
3) favorited: how many people "liked" the tweet. check if null.
4) text: text of the tweet, check if empty.
5) language: language of the tweet. Useful for translating later. check if null.
6) reply: user that the tweet replied to. check if null. Absence does not mean an error but this will give a sense of how many are replies.
7) retweet count: how many times the tweet was retweeted by others. Should be int.
8) witheld in countries: list of countries banning this tweet. check if null.

%%R
multiplot <- function(..., plotlist=NULL, file, cols=1, layout=NULL) {
  require(grid)

  # Make a list from the ... arguments and plotlist
  plots <- c(list(...), plotlist)

  numPlots = length(plots)

  # If layout is NULL, then use 'cols' to determine layout
  if (is.null(layout)) {
    # Make the panel
    # ncol: Number of columns of plots
    # nrow: Number of rows needed, calculated from # of cols
    layout <- matrix(seq(1, cols * ceiling(numPlots/cols)),
                    ncol = cols, nrow = ceiling(numPlots/cols))
  }

 if (numPlots==1) {
    print(plots[[1]])

  } else {
    # Set up the page
    grid.newpage()
    pushViewport(viewport(layout = grid.layout(nrow(layout), ncol(layout))))

    # Make each plot, in the correct location
    for (i in 1:numPlots) {
      # Get the i,j matrix positions of the regions that contain this subplot
      matchidx <- as.data.frame(which(layout == i, arr.ind = TRUE))

      print(plots[[i]], vp = viewport(layout.pos.row = matchidx$row,
                                      layout.pos.col = matchidx$col))
    }
  }
}
